二分查找不是“每次算一次 `mid`”，而是：在一个有序（或可判断单调性）的范围内，**每轮排除一半不可能包含答案的区间**。

写二分时先确定两件事：

1. 答案可能落在哪个区间；
2. 每次判断后，`mid` 能不能被排除。

后面的 `r` 初值、循环条件、边界更新都由这两点决定，不能从不同模板里拆开拼。

## 二分查找和双指针

|  | 二分查找 | 双指针 |
| --- | --- | --- |
| 核心动作 | 一次排除约一半范围 | 一次移动一格或少量位置 |
| 常见复杂度 | `O(log n)` | 通常为 `O(n)` |
| 前提 | 有序性或单调性 | 常用于有序数组、窗口、链表等 |
| 典型场景 | 查值、找边界、找最小可行值 | 两数之和、滑动窗口、链表快慢指针 |

下面这种写法虽然有 `mid`，但不是二分：

```python
if nums[mid] < target:
    l += 1
else:
    r -= 1
```

它每轮只缩小一个位置，最坏仍是 `O(n)`。二分更新应直接越过半边：`l = mid + 1` 或 `r = mid - 1`；若 `mid` 仍可能是答案，则保留它，写成 `r = mid`。

## 先记住：区间和不变量

所谓不变量，就是循环过程中始终成立的一句话：**答案仍在当前搜索区间内**。

例如在 `lower_bound` 中，搜索区间为 `[l, r)`，并维持：

> 第一个 `>= target` 的位置始终在 `[l, r)` 中。

若 `nums[mid] < target`，`mid` 和左边都不可能是答案，令 `l = mid + 1`；否则 `mid` 可能就是答案，不能丢掉它，令 `r = mid`。当 `l == r` 时，区间收缩为一个位置，答案就是 `l`。

## 模板一：左闭右开 `[l, r)`——找插入位置 / 边界

这是最推荐优先掌握的模板。`r = len(nums)` 表示右端是**允许返回、但不能访问的哨兵位置**，所以结果可以等于 `len(nums)`。

```python
def lower_bound(nums, target):
    """返回第一个 >= target 的下标；不存在时返回 len(nums)。"""
    l, r = 0, len(nums)
    while l < r:
        mid = l + (r - l) // 2
        if nums[mid] < target:
            l = mid + 1
        else:
            r = mid
    return l
```

只改一个判断，就得到第一个 `> target` 的位置：

```python
def upper_bound(nums, target):
    """返回第一个 > target 的下标；不存在时返回 len(nums)。"""
    l, r = 0, len(nums)
    while l < r:
        mid = l + (r - l) // 2
        if nums[mid] <= target: #修改点
            l = mid + 1
        else:
            r = mid
    return l
```

| 要找的位置 | `mid` 满足条件时怎么做 | 不满足时怎么做 |
| --- | --- | --- |
| 第一个 `>= x` | `nums[mid] < x`：`l = mid + 1` | `r = mid` |
| 第一个 `> x` | `nums[mid] <= x`：`l = mid + 1` | `r = mid` |

## 模板二：闭区间 `[l, r]`——直接查找某个值

这里的 `l`、`r` 都是实际下标，因此 `r = len(nums) - 1`。循环继续条件是 `l <= r`，这样单元素区间也会被检查。

```python
def binary_search(nums, target):
    l, r = 0, len(nums) - 1
    while l <= r:
        mid = l + (r - l) // 2
        if nums[mid] == target:
            return mid
        if nums[mid] < target:
            l = mid + 1
        else:
            r = mid - 1
    return -1
```

`nums[mid] != target` 时，`mid` 已被证明不是答案，故必须用 `mid + 1` 或 `mid - 1` 把它排除；否则可能死循环。

## 模板三：闭区间收缩 `[l, r]`——找最小值 / 临界点

有些题不需要“查不到返回 -1”，而是确定答案一定在数组的某个真实下标中。此时让区间收缩到一个点即可：

```python
l, r = 0, len(nums) - 1
while l < r:
    mid = l + (r - l) // 2
    if 条件说明答案在右半边且不可能是 mid:
        l = mid + 1
    else:  # mid 仍可能是答案
        r = mid
return l
```

关键仍是 `mid` 能否排除：能排除才 `l = mid + 1`；不能排除就保留为 `r = mid`。

## 34. 在排序数组中查找元素的第一个和最后一个位置

给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 `target`，返回 `[-1, -1]`。

### 思路：二分找左右边界

`lower_bound(target)` 找到左边界；`upper_bound(target) - 1` 找到右边界。先验证左边界确实指向 `target`，即可处理目标不存在、空数组和边界越界。

```python
class Solution:
    def searchRange(self, nums: List[int], target: int) -> List[int]:
        def lower_bound(x):
            l, r = 0, len(nums)
            while l < r:
                mid = l + (r - l) // 2
                if nums[mid] < x:# 第一个≥x的
                    l = mid + 1
                else:
                    r = mid
            return l

        def upper_bound(x):
            l, r = 0, len(nums)
            while l < r:
                mid = l + (r - l) // 2
                if nums[mid] <= x: # 第一个＞x的
                    l = mid + 1
                else:
                    r = mid
            return l

        left = lower_bound(target)
        if left == len(nums) or nums[left] != target:
            return [-1, -1]
       	right = upper_bound(target) - 1
        return [left, right]
```

- 这里的`right`也可以由`lower_bound(target+1)-1`实现

Python 也可直接使用标准库：

```python
from bisect import bisect_left, bisect_right

left = bisect_left(nums, target)
right = bisect_right(nums, target) - 1
answer = [left, right] if left <= right else [-1, -1]
```

## 33. 搜索旋转排序数组

给你 **旋转后** 的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1` 。

- 所谓旋转，是指对严格升序的原数组（无重复值），截取某一段`nums[:k]`移到了数组末尾，k可以为0，即未旋转。

### 思路一：先找旋转点，再在一侧二分

1. 先用二分找到旋转数组里的最小值下标 `pivot`
2. 判断 `target` 应该在最小值右边那段，还是左边那段
3. 再对目标区间做一次普通二分

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        n = len(nums)

        left,right = 0,n-1
        while left<right:
            mid = left+(right-left)//2
            # 默认升序，如果中间的更大说明被移动了
            if nums[mid]>nums[right]:
                left = mid+1
            else:
                right = mid #这里很显然如果mid<=right，就自然把高值缩短到mid
        # 本质是找到当前序列里的最小值
        pivot = left
        # 题目的旋转使得nums被分割成两段升序序列
        # 前者要更大一些，判断target在哪一段
        if nums[pivot]<=target<=nums[-1]:
            low,high = pivot,n-1 #在第二段
        else:
            low,high = 0,pivot-1 #在第一段
        # 这里就是常规二分了
        while low<=high:
            m = low+(high-low)//2
            if nums[m]==target:
                return m
            elif nums[m]<target:
                low = m+1
            else:
                high = m-1
        # 注意上面的循环条件多了一个等号，为的是多走一次循环（即low=high）的时候
        # 此时m=low=high，判断是否等于target，从而返回
        # 所以也等价于 return -1 if nums[low]!=target else low
        return -1
```

### 思路二：判断哪段有序

数组无重复值时，`mid` 的一侧一定有序。每轮先判断哪一侧有序，再看 `target` 是否落在这段有序区间内；这样只保留可能包含目标的一半。

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        l, r = 0, len(nums) - 1

        while l <= r:
            mid = l + (r - l) // 2
            if nums[mid] == target:
                return mid

            if nums[l] <= nums[mid]:  # 左半段有序
                if nums[l] <= target < nums[mid]:
                    r = mid - 1
                else:
                    l = mid + 1
            else:  # 右半段有序
                if nums[mid] < target <= nums[r]:
                    l = mid + 1
                else:
                    r = mid - 1

        return -1
```

## 写题检查

- 区间是 `[l, r)` 还是 `[l, r]`？
- 循环结束时，答案应当还在区间里，还是区间已经为空？
- 当前分支中，`mid` 还有可能是答案吗？有则保留 `mid`，否则排除它。
- `r = len(nums)` 时，绝不能访问 `nums[r]`。

一句话：**不要单看 `r` 的初值；区间定义、循环条件、更新规则和返回值必须是一套。**
