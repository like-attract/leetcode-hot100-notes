贪心算法的核心是：

> 每一步都做一个“当前看起来最优”的选择，并且希望这些局部最优选择最终能得到全局最优解。

但贪心不是“拍脑袋选最大/最小”，关键是：**这个局部选择必须能证明不会破坏最终最优解**。

常见特征：

- 每一步只需要维护一个“当前最优状态”
- 不需要回溯
- 通常时间复杂度是 `O(n)`
- 常见于：最值问题、可达性问题、区间问题、分割问题

下面结合 LeetCode Hot100 里的 4 道贪心题来讲：

- 121. 买卖股票的最佳时机
- 55. 跳跃游戏
- 45. 跳跃游戏 II
- 763. 划分字母区间

## 一、121. 买卖股票的最佳时机

题目大意：

给定每天股价，只能买卖一次，求最大利润。

例如：

```text
prices = [7, 1, 5, 3, 6, 4]
```

答案是：

```text
6 - 1 = 5
```

### 贪心思路

对于某一天卖出来说，最优策略一定是：

> 在这一天之前的最低价格买入。

所以遍历数组时，维护两个变量：

- `minPrice`：当前遇到的最低价格
- `maxProfit`：当前最大利润

每一天都计算：

```text
当前价格 - 历史最低价
```

然后更新最大利润。

### 代码

```python
def maxProfit(prices):
    minPrice = float("inf")
    maxProfit = 0

    for price in prices:
        minPrice = min(minPrice, price)
        maxProfit = max(maxProfit, price - minPrice)

    return maxProfit
```

### 为什么是贪心？

因为对于每个卖出日来说，买入价越低越好。

所以每一步只需要保留：

```text
历史最低价
```

这就是局部最优选择。

## 二、55. 跳跃游戏

题目大意：

数组中每个元素表示当前位置最多能跳多远，判断能不能跳到最后一个位置。

例如：

```text
nums = [2, 3, 1, 1, 4]
```

答案是 `true`。

因为：

```text
0 -> 1 -> 4
```

可以到达最后。

### 贪心思路

维护一个变量：

```text
farthest
```

表示当前能够到达的最远位置。

遍历数组时：

```text
farthest = max(farthest, i + nums[i])
```

如果某个位置 `i` 已经超过了 `farthest`，说明这个位置根本到不了，直接返回 `false`。

### 代码

```python
def canJump(nums):
    farthest = 0
    n = len(nums)

    for i in range(n):
        if i > farthest:
            return False

        farthest = max(farthest, i + nums[i])

        if farthest >= n - 1:
            return True

    return True
```

### 为什么是贪心？

因为我们不关心具体跳到哪个位置，只关心：

> 当前能到达的最远位置是多少。

如果当前最远位置已经能覆盖终点，那么一定可以到达。

## 三、45. 跳跃游戏 II

题目大意：

和 55 类似，但这题要求：

> 跳到最后一个位置的最少步数。

例如：

```text
nums = [2, 3, 1, 1, 4]
```

答案是：

```text
2
```

因为：

```text
0 -> 1 -> 4
```

### 贪心思路

这题可以理解为：

> 每次跳跃都扩展当前能到达的范围。

维护三个变量：

- `jumps`：已经跳了几次
- `curEnd`：当前这一跳能到达的边界
- `farthest`：下一跳最远能到哪里

遍历数组时，不断更新 `farthest`。

当走到当前跳跃边界 `curEnd` 时，说明必须再跳一次。

### 代码

```python
def jump(nums):
    n = len(nums)
    jumps = 0
    curEnd = 0
    farthest = 0

    for i in range(n - 1):
        farthest = max(farthest, i + nums[i])

        if i == curEnd:
            jumps += 1
            curEnd = farthest

    return jumps
```

### 为什么是贪心？

在当前位置能跳到的范围内，我们不需要马上决定跳到哪一个点，只需要知道：

> 从当前范围内下一步最远能到哪里。

这就像一层一层向外扩展：

```text
第 0 跳能到的范围
第 1 跳能到的范围
第 2 跳能到的范围
...
```

每次到达当前范围边界，就必须增加一次跳跃。

## 四、763. 划分字母区间

题目大意：

给定一个字符串，把它划分成尽可能多的片段，使得同一个字母只出现在一个片段中。

例如：

```text
s = "ababcbacadefegdehijhklij"
```

答案是：

```text
[9, 7, 8]
```

对应划分：

```text
"ababcbaca"
"defegde"
"hijhklij"
```

### 贪心思路

关键是：

> 如果一个字符出现在当前片段中，那么这个片段必须至少覆盖它最后一次出现的位置。

所以先记录每个字符最后出现的位置：

```python
last = {char: index for index, char in enumerate(s)}
```

然后从左到右遍历字符串。

维护：

- `start`：当前片段的起点
- `end`：当前片段必须延伸到的最远位置

当遍历到某个字符 `c` 时：

```python
end = max(end, last[c])
```

如果当前位置 `i == end`，说明当前片段已经可以切开了。

### 代码

```python
def partitionLabels(s):
    last = {char: i for i, char in enumerate(s)}

    res = []
    start = 0
    end = 0

    for i, char in enumerate(s):
        end = max(end, last[char])

        if i == end:
            res.append(end - start + 1)
            start = i + 1

    return res
```

### 为什么是贪心？

因为每个片段都想尽量短。

但是一个字符如果后面还会出现，当前片段就不能提前结束。

所以每一步都维护：

```text
当前片段必须覆盖到的最远位置
```

一旦当前位置到达这个最远位置，就说明可以安全切分。

| ---- | ------------ | ---------------------------- | -------------------------- |
| 121  | 最大利润     | 历史最低价                   | 每天用历史最低价买入       |

## 六、总结一句话

贪心算法的本质是：

> 在每一步都做出当前最优决策，并且这个决策不会让后面的结果变差。

这四道题的贪心点分别是：

```text
121：卖股票时，买入价越低越好
55：只要能到达的最远距离够远，就能继续扩展
45：每次跳跃尽量覆盖最远范围
763：片段必须覆盖其中所有字符的最后出现位置
```

判断一个题能不能用贪心，可以问自己两个问题：

1. 当前这一步的局部最优选择是什么？
2. 这个局部最优选择为什么不会破坏最终答案？

如果这两个问题能讲清楚，基本就可以尝试贪心了。
